# Быстрый старт

Это руководство поможет вам создать новый проект на BAD архитектуре и запустить его за несколько минут.

## Предварительные требования

Убедитесь, что в вашей системе установлены:

- [Node.js](https://nodejs.org/) (версия 18 или выше)
- [pnpm](https://pnpm.io/) — рекомендуемый менеджер пакетов (вы можете использовать npm или yarn, но проект заточен под pnpm)
- [NestJS CLI](https://docs.nestjs.com/cli/overview) (устанавливается глобально: `npm i -g @nestjs/cli`) — потребуется для работы генератора роутов внутри проекта

## Установка BAD CLI

Для создания нового проекта используется пакет `bad-fockarch`, который предоставляет команду `bad`

### Глобальная установка

```bash
npm install -g bad-fockarch
# или
pnpm add -g bad-fockarch
```

## Создание нового проекта

Команда `bad create` создаёт новый проект на основе шаблона BAD. Синтаксис:

```bash
bad create <имя_проекта> [путь_размещения] [менеджер_пакетов]
```

- `<имя_проекта>` — обязательный параметр, название вашего приложения (будет использовано как имя папки).
- `[путь_размещения]` — опциональный путь, куда будет создан проект. По умолчанию — текущая папка.
- `[менеджер_пакетов]` — опционально, какой пакетный менеджер использовать для установки зависимостей (`npm`, `pnpm`, `yarn`). По умолчанию `pnpm`.

**Примеры:**

```bash
# Создать проект my-app в текущей папке, используя pnpm
bad create my-app

# Создать проект awesome-api в папке ./projects, используя npm
bad create awesome-api ./projects npm
```

После выполнения команды вы увидите сообщение об успешном создании проекта. Перейдите в созданную папку:

```bash
cd my-app
```

Все зависимости уже будут установлены (команда `create` автоматически запускает установку). Проект готов к настройке.

> **Альтернативный способ:** Вы также можете склонировать репозиторий шаблона вручную:
>
> ```bash
> git clone https://github.com/Lazy-And-Focused/BAD-template.git my-app
> cd my-app
> pnpm install
> ```

## Настройка окружения

1. В корне проекта скопируйте файл с примером переменных окружения:

    ```bash
    cp .env.example .env
    ```

2. Откройте файл `.env` в любом редакторе и заполните необходимые значения:

   - `PORT` — порт, на котором будет работать приложение (например, `8080`).
   - `DATABASE_URL` — строка подключения к вашей базе данных (если вы используете Prisma).
   - `SESSION_SECRET` и `HASH_KEY` — сгенерируйте случайные строки (можно использовать `openssl rand -hex 32`).
   - `CLIENT_URL` — адрес вашего фронтенд-приложения для настройки CORS.
   - Остальные переменные можно оставить по умолчанию или изменить позже.

## Запуск приложения

Теперь всё готово для запуска. В зависимости от выбранного менеджера пакетов используйте соответствующие команды.

### Другие полезные команды

- `pnpm run build` — сборка проекта для production.
- `pnpm run prod` — запуск собранного production-приложения.
- `pnpm run start:dev` — запуск в режиме разработки без авто-перезапуска.
- `pnpm run lint` — проверить код линтером.
- `pnpm run format` — отформатировать код с Prettier.

## Проверка работы

После успешного запуска в консоли вы увидите сообщение, что приложение слушает порт (например, `http://localhost:8080`). Откройте браузер и перейдите по адресу:

```
http://localhost:8080/api/docs
```

Вы должны увидеть страницу Swagger UI с автоматически сгенерированной документацией вашего API. Это означает, что проект работает корректно.

## Что дальше?

Теперь вы можете приступить к созданию собственных роутов. Для этого используйте встроенный генератор:

```bash
nest g badp user
```

Эта команда создаст новый роут `users` со всеми необходимыми файлами внутри `src/routes/`. Подробнее об этом читайте в разделе **«Работа с роутами»**.
