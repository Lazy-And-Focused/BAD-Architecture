# Структура рабочей области (Workspace)

Корневая директория проекта (workspace) содержит все конфигурационные файлы, инструменты, зависимости и документацию. Она организована стандартным для NestJS-приложений образом, но с некоторыми дополнениями, характерными для BAD архитектуры. Понимание назначения каждого элемента поможет эффективно настраивать проект и избегать ошибок.

> **Обозначения:**
>
> - `?` — необязательный файл/папка (может отсутствовать, если не нужен).
> - Комментарии в скобках `#` даны для пояснения и не являются частью структуры.

```
└── {workFolder}/                     # Корневая папка проекта (название может быть любым)
    ├── .dockerignore                  # Список исключений для Docker
    ├── .env                           # Основной файл переменных окружения (не в репозитории)
    ├── .env.{envName}                  # Файлы окружения для разных сред (production, test и т.д.) (?)
    ├── .gitignore                      # Список файлов, игнорируемых Git
    ├── .prettierrc.mjs                 # Конфигурация Prettier
    ├── docker-compose.yaml              # Docker Compose для локального запуска (БД и т.д.)
    ├── dockerfile                       # Инструкция сборки Docker-образа приложения
    ├── eslint.config.mjs                # Конфигурация ESLint
    ├── LICENSE                          # Лицензия проекта (MIT по умолчанию)
    ├── nest-cli.json                    # Конфигурация NestJS CLI (лучше не трогать)
    ├── package.json                     # Манифест Node.js-приложения (зависимости, скрипты)
    ├── pnpm-lock.yaml                    # Lock-файл pnpm (обеспечивает воспроизводимость)
    ├── README.md                         # Основная документация проекта
    ├── tsconfig.build.json                # Настройки TypeScript для сборки (лучше не трогать)
    ├── tsconfig.build.tsbuildinfo          # Служебный файл TypeBuildInfo (генерируется автоматически)
    ├── tsconfig.json                       # Основная конфигурация TypeScript (лучше не трогать)
    ├── test/                               # Папка с e2e-тестами и конфигурацией Jest
    │   ├── app.e2e-spec.ts                  # Пример e2e-теста
    │   ├── jest-e2e.json                     # Конфигурация Jest для e2e
    │   └── jest.json                          # Основная конфигурация Jest для модульных тестов
    └── src/                                 # Исходный код приложения (подробно описан в отдельном документе)
```

## Пояснения по элементам корневой папки

| Элемент | Назначение | Возможность изменения |
| ------- | ---------- | ---------------------- |
| `.dockerignore` | Указывает, какие файлы не копировать в Docker-образ. | Можно изменять под нужды контейнеризации. |
| `.env` | Переменные окружения для текущей среды (например, локальной). Создаётся из `.env.example`. | Можно и нужно редактировать (но не хранить в репозитории). |
| `.env.{envName}` | Переменные для конкретных сред (`.env.production`, `.env.test`). | Можно создавать и изменять по необходимости. |
| `.gitignore` | Исключает временные и служебные файлы из репозитория. | Можно дополнять своими исключениями. |
| `.prettierrc.mjs` | Настройки форматирования кода. | Можно изменять под стиль команды. |
| `docker-compose.yaml` | Описание сервисов для локальной разработки (база данных, кэш и т.д.). | Можно адаптировать под свой compose-стек. |
| `dockerfile` | Инструкция сборки образа приложения. | Можно изменять под требования деплоя. |
| `eslint.config.mjs` | Настройки линтинга. | Можно настраивать правила под проект. |
| `LICENSE` | Текст лицензии. | Менять, если требуется другая лицензия. |
| `nest-cli.json` | Конфигурация NestJS CLI (пути, компилятор). | **Лучше не трогать** без крайней необходимости. |
| `package.json` | Зависимости, скрипты, метаданные проекта. | Можно добавлять/удалять зависимости, менять скрипты. |
| `pnpm-lock.yaml` | Фиксирует версии установленных пакетов. | Не редактировать вручную (автоматический). |
| `README.md` | Документация проекта. | Можно и нужно дополнять описанием своего проекта. |
| `tsconfig.build.json` | Настройки TypeScript для production-сборки. | **Лучше не трогать** (наследуется от `tsconfig.json`). |
| `tsconfig.build.tsbuildinfo` | Служебный файл для инкрементальной сборки. | Игнорировать (генерируется автоматически). |
| `tsconfig.json` | Основные настройки TypeScript (включая алиасы путей). | **Лучше не трогать** без понимания последствий. |
| `test/` | Модульные и e2e-тесты. | Можно изменять и дополнять тесты. |
| `src/` | Весь исходный код приложения. | Основное место работы разработчика. Подробная структура описана в [Структура проекта (src)](project-structure.md). |

## Важные замечания

- **Файлы `.env*` никогда не должны попадать в репозиторий** — они уже добавлены в `.gitignore`. Всегда используйте `.env.example` как шаблон для новых переменных.
- **Lock-файлы** (`pnpm-lock.yaml`) обязательно должны храниться в репозитории, чтобы обеспечить одинаковые версии зависимостей у всех разработчиков.
- **Папки `node_modules/` и `dist/`** автоматически игнорируются Git и не должны попадать в репозиторий.
- **Файлы конфигурации TypeScript и NestJS** изменяйте только при полном понимании их работы, так как они критически влияют на компиляцию и сборку.

Данная структура корневой папки является стандартной для BAD-проектов и не требует ручного создания — она генерируется автоматически при использовании команды `bad create`.
