# Введение в BAD Архитектуру

**BAD** (Backend After Drinking) — это архитектурный шаблон и набор
инструментов для создания backend-приложений на NestJS. Несмотря на
шутливую расшифровку названия, BAD предлагает строгий и продуманный
подход к структурированию кода, который делает разработку быстрой, а
проект — легким в поддержке и масштабировании.

## Философия

Архитектура BAD родилась из практического опыта и желания иметь под
рукой надежный шаблон, который берет на себя рутину настройки и
организации файлов. Основная идея — дать разработчику жёсткий каркас,
внутри которого уже есть всё необходимое для старта, и при этом
оставить свободу для реализации конкретной бизнес-логики.

BAD — это проект команды **LAF (Lazy And Focused)**. Наше кредо: "Лень
подчинила себе процессы, пока мы фокусировались на обязанностях".
Поэтому BAD создан, чтобы вы могли фокусироваться на решении задач, а
не на том, как назвать папку или куда положить файл.

## Ключевые возможности

BAD предоставляет не просто структуру папок, а целый рабочий процесс:

- **Готовая структура проекта.** Чёткое разделение на версии API,
  роуты, сервисы, утилиты и другие слои приложения.
- **Кастомный генератор кода.** С помощью команд `nest g bad` вы
  создаёте новые роуты со всем необходимым набором файлов (контроллер,
  сервис, модуль, DTO) за секунды.
- **Интеграция лучших практик.** В шаблон уже встроены и настроены:
  - Swagger для автоматической документации API.
  - Sentry для отслеживания ошибок.
  - Механизмы троттлинга и кэширования.
  - Passport.js для гибкой аутентификации.
  - Готовые Dockerfile и docker-compose.
- **Единый стиль кода.** Архитектура опирается на стандарты написания
  кода команды LAF, что гарантирует чистоту, читаемость и
  предсказуемость кодовой базы.

BAD подходит для проектов любого масштаба: от небольшого API для
личного блога до сложной корпоративной системы с несколькими версиями
и микросервисами.
