# BAD Architecture

**Backend After Drinking** — это архитектурный шаблон и набор инструментов для создания backend-приложений на NestJS. Проект разработан командой LAF (Lazy And Focused) и сочетает в себе строгую структуру, гибкость и автоматизацию рутинных задач.

## Философия

Название родилось из шутливого отношения к сложности бэкенд-разработки, но за ним стоит серьёзный подход. Мы считаем, что код должен быть единообразным, предсказуемым и лёгким в поддержке. BAD берёт на себя настройку проекта, организацию файлов и генерацию шаблонного кода, чтобы вы могли сосредоточиться на бизнес-логике.

## Ключевые возможности

- **Готовая структура проекта** с чётким разделением на версии API, роуты, сервисы и утилиты.
- **Кастомный CLI** (`bad create`), который создаёт новый проект с установленными зависимостями за секунду.
- **Генератор роутов** (`nest g bad`), автоматически создающий контроллер, сервис, модуль, DTO и файл маршрутов.
- **Поддержка версионности API** — изолированные папки `v1/`, `v2/` позволяют развивать API без поломки обратной совместимости.
- **Интеграция лучших практик** из коробки:
  - Swagger для автодокументации
  - Sentry для отслеживания ошибок
  - Throttler и Cache Manager для производительности
  - Passport.js для аутентификации
  - Docker для контейнеризации
- **Единый стиль кода** на основе стандартов команды LAF, включая правила именования, форматирования и линтинга.

## Для кого этот проект

BAD подойдёт командам и индивидуальным разработчикам, которые ценят порядок в коде и хотят быстро стартовать с надёжной архитектурой. Он одинаково полезен как для небольших прототипов, так и для крупных корпоративных систем.
